# excel-to-json — 用户故事

## 1. 核心用户故事

### US-01: 单文件转换
作为开发者，我希望将一个 Excel 文件转换为 JSON，以便在程序中使用结构化数据。

**验收标准：**
- 支持 .xlsx, .xls, .xlsb, .ods, .csv 格式
- 输出为有效的 JSON 文件
- 保留原始数据类型（字符串、数字、布尔、日期）

### US-02: 目录批量处理
作为开发者，我希望指定一个目录，自动发现并转换目录中所有 Excel 文件。

**验收标准：**
- 递归遍历子目录（可通过参数控制）
- 每个文件输出对应的 JSON 文件，保持目录结构
- 每个文件的转换进度通过 stdout 通知

### US-03: 配置文件驱动映射
作为开发者，我希望通过配置文件定义列名映射规则，以便将 Excel 的中文列名转换为英文 JSON key。

**验收标准：**
- 支持 TOML 格式的配置文件
- 支持列重命名（"姓名" → "name"）
- 支持列排除
- CLI 参数覆盖配置文件中的设置

### US-04: 进程间通信
作为系统集成者，我希望通过 stdout 获取结构化的转换进度信息，以便与其他进程协作。

**验收标准：**
- 每行输出一个完整的 JSON 消息
- 消息类型包括：progress（进度）、done（完成）、error（错误）
- stderr 用于诊断日志，stdout 仅用于结构化消息

## 2. 非功能需求

- 单个 100MB 以内的 Excel 文件应在 10 秒内完成转换
- 工具应能在 Linux、macOS 上运行
- 错误信息应清晰，指导用户解决问题
