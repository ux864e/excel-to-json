# excel-to-json — 用户故事

## 1. 核心用户故事

### US-01: 单文件转换
作为开发者，我希望将一个 Excel 文件转换为 JSON 配置，以便在程序中使用结构化数据。

**验收标准：**
- 支持 .xlsx, .xls, .xlsb, .ods 格式
- 输出为有效的 JSON 文件
- 保留原始数据类型（字符串、数字、布尔、日期）
- 每个 sheet 产生一个独立配置文件

### US-02: 目录批量处理
作为开发者，我希望指定一个目录，自动发现并转换目录中所有 Excel 文件。

**验收标准：**
- 递归遍历子目录（可通过参数控制）
- 每个 sheet 输出对应的 JSON 配置文件
- 转换结果通过 stdout 单行 JSON 摘要通知
- 单文件错误不中止整批转换

### US-03: 配置文件驱动映射
作为开发者，我希望通过配置文件定义列名映射规则，以便将 Excel 的中文列名转换为英文 JSON key。

**验收标准：**
- 支持 TOML 格式的配置文件
- 支持列重命名（"姓名" → "name"）
- 支持列排除
- CLI 参数覆盖配置文件中的设置

### US-04: 进程间通信
作为系统集成者，我希望通过 stdout 获取结构化的转换结果信息，以便与其他进程协作。

**验收标准：**
- stdout 输出单行完整 JSON 摘要（status、files、errors、warnings）
- 空 errors/warnings 自动省略
- stderr 用于诊断日志，stdout 仅用于结构化消息
- 通过 stdin 管道接收 `outputDir` 配置

### US-05: 自描述 Excel 约定
作为配置维护者，我希望在 Excel 表格中内嵌元数据（configName、description），使得表格本身即配置定义，无需额外元数据文件。

**验收标准：**
- Row 0 Col B = configName（配置标识符）
- Row 1 Col B = description（描述）
- Row 2 = 字段定义（headers，Col A 固定为 `id`）
- Row 3 = 字段注释（跳过）
- Row 4+ = 数据行

### US-06: 数据质量保障
作为数据管理者，我希望工具能自动检测并报告数据质量问题，而不是静默忽略。

**验收标准：**
- `id` 列强制：必须为唯一无符号整数
- `//` 前缀行自动识别为注释跳过
- 重复 id 警告并跳过
- 无效 id 计数为 failedRows
- 每 config 独立报告 inputRows/validRows/skippedRows/failedRows

### US-07: 规范输出文件名
作为系统集成者，我希望输出文件名遵循统一的命名规范（lower kebab-case），无论 Excel 中 configName 使用什么大小写风格。

**验收标准：**
- camelCase configName 自动转为 kebab-case 文件名（如 `myConfigName` → `my-config-name.json`）
- 全小写 configName 保持不变
- configName 在 JSON 文件内容和摘要中原样保留

## 2. 非功能需求

- 单个 100MB 以内的 Excel 文件应在 10 秒内完成转换
- 工具应能在 Linux、macOS 上运行
- 错误信息应清晰，指导用户解决问题
- 执行结果可通过单行 JSON 解析判断成功/失败
