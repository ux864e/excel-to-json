# excel-to-json — 系统设计

## 1. 架构概览

```mermaid
flowchart TD
    MAIN[main.rs] -->|run()| LIB[lib.rs]
    LIB --> CLI[cli.rs: Args]
    LIB --> CFG[config.rs: Config]
    LIB --> WALK[walker.rs: collect_files]
    LIB --> CONV[converter.rs: convert_all]
    LIB --> OUT[output.rs: emit_results]

    WALK -->|ExcelFile[]| CONV
    CONV -->|Raw Rows| MAP[mapping.rs: apply_mapping]
    MAP -->|JSON Values| CONV
    CONV -->|ConversionResult[]| OUT
    OUT -->|JSON Files| DISK[(Output Dir)]
    OUT -->|IPC| STDOUT>stdout]

    style MAIN fill:#2DBE9B,color:#FFFFFF
    style LIB fill:#2DBE9B,color:#FFFFFF
```

## 2. 模块职责

### 2.1 `main.rs` — 二进制入口
- 调用 `excel_to_json::run()`
- 错误时打印到 stderr 并 exit(1)

### 2.2 `lib.rs` — 库入口
- 声明所有模块
- 实现 `run()` 编排函数

### 2.3 `cli.rs` — 命令行参数
- clap derive 定义 `Args` 结构体
- `resolve_config_path()` 方法

### 2.4 `config.rs` — 配置加载
- `Config` 运行时配置
- `ConfigFile` TOML 文件结构
- `MappingConfig` 映射规则
- CLI 参数覆盖配置文件

### 2.5 `walker.rs` — 目录遍历
- 发现支持的 Excel 文件
- 返回 `ExcelFile` 列表

### 2.6 `converter.rs` — 核心转换
- calamine 读取 Excel/CSV
- 提取表头和数据行
- 调用 `mapping` 模块进行字段转换

### 2.7 `mapping.rs` — 字段映射
- 列重命名（column_map）
- 列排除（exclude_columns）
- 嵌套路径（nested_paths）
- 单元格值 JSON 化

### 2.8 `output.rs` — 输出
- JSON 文件写入
- stdout IPC 消息（progress/done/error）
- stderr tracing 日志

## 3. 数据模型

### 3.1 IpcMessage（stdout IPC）

| 类型 | 字段 | 说明 |
|------|------|------|
| progress | file, status | 开始转换某文件 |
| done | file, rows | 文件转换完成，含行数 |
| error | file, message | 转换失败 |

### 3.2 Config

| 字段 | 类型 | 来源 |
|------|------|------|
| input_dir | PathBuf | CLI (default ".") |
| output_dir | PathBuf | TOML or CLI (default "./output") |
| recursive | bool | TOML or CLI (default true) |
| pretty | bool | TOML or CLI (default false) |
| mapping | MappingConfig | TOML only |

### 3.3 MappingConfig

| 字段 | 类型 | 说明 |
|------|------|------|
| column_map | HashMap<String, String> | 列重命名映射 |
| exclude_columns | Vec<String> | 排除列 |
| nested_paths | HashMap<String, String> | 嵌套路径映射 |
