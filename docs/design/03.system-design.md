# excel-to-json — 系统设计

## 1. 架构概览

```mermaid
flowchart TD
    MAIN[main.rs] -->|run()| LIB[lib.rs]
    LIB --> CLI[cli.rs: Args]
    LIB --> CFG[config.rs: Config + StdinInput]
    LIB --> WALK[walker.rs: walk_dir]
    LIB --> CONV[converter.rs: convert_all]
    LIB --> OUT[output.rs: emit_results + emit_summary]

    STDIN[stdin pipe] -->|outputDir| CFG
    TOML[excel-to-json.toml] -->|Base| CFG
    CLI -->|Override| CFG

    WALK -->|ExcelFile[]| CONV
    CONV -->|parse_sheet_with_meta| SHEET[Sheet Parser]
    SHEET -->|validate_config_name| NAME[Config Name Validator]
    SHEET -->|Raw Rows| MAP[mapping.rs: apply_mapping]
    MAP -->|JSON Values| CONV
    CONV -->|ConversionResult[]| OUT
    OUT -->|camel_to_kebab| FNAME[Filename Normalizer]
    FNAME -->|JSON Files| DISK[(Output Dir)]
    OUT -->|Summary| STDOUT>stdout]

    style MAIN fill:#2DBE9B,color:#FFFFFF
    style LIB fill:#2DBE9B,color:#FFFFFF
    style STDIN fill:#7B68EE,color:#FFFFFF
```

## 2. 模块职责

### 2.1 `main.rs` — 二进制入口
- 调用 `excel_to_json::run()`
- 错误时打印到 stderr 并 exit(1)

### 2.2 `lib.rs` — 库入口
- 声明所有模块
- 实现 `run()` 编排函数：加载配置 → 遍历文件 → 转换 → 输出
- 处理 stdin 管道输入

### 2.3 `cli.rs` — 命令行参数
- clap derive 定义 `Args` 结构体
- `resolve_config_path()` 方法（在 input_dir 下查找 excel-to-json.toml）

### 2.4 `config.rs` — 配置加载
- `Config` 运行时配置（合并 TOML + CLI + stdin）
- `ConfigFile` TOML 文件结构
- `StdinInput` stdin 管道输入（仅 `outputDir`）
- `MappingConfig` 映射规则
- `read_stdin_config()` 尝试从 stdin 读取配置

### 2.5 `walker.rs` — 目录遍历
- 发现支持的 Excel 文件（.xlsx/.xls/.xlsb/.ods）
- 记录相对路径和 stem
- 返回 `ExcelFile` 列表

### 2.6 `converter.rs` — 核心转换
- calamine 读取 Excel，逐 sheet 处理
- `parse_sheet_with_meta()`: 按元数据约定提取 configName、description、headers
- `validate_config_name()`: 校验 configName 格式
- `comment_column_mask()`: 识别 `//` / `#` 前缀的参考列
- `cell_to_string()`: 单元格值转字符串
- `parse_id()`: 解析 u64 id
- 调用 `mapping` 模块进行字段转换
- 统计 inputRows/validRows/skippedRows/failedRows
- `ConversionSummary` 收集结果和错误

### 2.7 `mapping.rs` — 字段映射
- 列重命名（column_map）
- 列排除（exclude_columns，id 列始终保留）
- 嵌套路径（nested_paths）
- 单元格值 JSON 化

### 2.8 `output.rs` — 输出
- `emit_results()`: 写入 JSON 文件到磁盘（含目录创建）
- `emit_summary()`: stdout 单行 JSON 摘要
- `camel_to_kebab()`: configName → kebab-case 文件名转换
- `Summary`/`FileSummary`/`ErrorEntry`/`WarningEntry` 数据结构

## 3. 数据模型

### 3.1 Summary（stdout 摘要）

| 字段 | 类型 | 说明 |
|------|------|------|
| status | String | "success" 或 "error" |
| files | Vec\<FileSummary\> | 成功输出的文件列表 |
| errors | Vec\<ErrorEntry\> | 错误列表（空时省略） |
| warnings | Vec\<WarningEntry\> | 警告列表（空时省略） |

### 3.2 FileSummary

| 字段 | JSON Key | 类型 | 说明 |
|------|----------|------|------|
| config_name | configName | String | 来自 Excel 的 configName（保留原值） |
| path | path | String | 输出文件名（kebab-case + .json） |
| description | description | String | 来自 Excel 的描述 |
| valid_rows | validRows | usize | 成功转换行数 |
| input_rows | inputRows | usize | 总数据行数 |
| skipped_rows | skippedRows | usize | 注释行 + 重复 id |
| failed_rows | failedRows | usize | 无效 id 行数 |

### 3.3 Config

| 字段 | 类型 | 来源 |
|------|------|------|
| input_dir | PathBuf | CLI (default ".") |
| output_dir | PathBuf | TOML or CLI or stdin (default "./output") |
| recursive | bool | TOML or CLI (default true) |
| pretty | bool | TOML or CLI (default false) |
| mapping | MappingConfig | TOML only |

### 3.4 MappingConfig

| 字段 | 类型 | 说明 |
|------|------|------|
| column_map | HashMap\<String, String\> | 列重命名映射 |
| exclude_columns | Vec\<String\> | 排除列（id 始终保留） |
| nested_paths | HashMap\<String, String\> | 嵌套路径映射 |

### 3.5 StdinInput

| 字段 | JSON Key | 类型 | 说明 |
|------|----------|------|------|
| output_dir | outputDir | PathBuf | stdin 管道传入的输出目录绝对路径 |

### 3.6 输出文件命名规则

configName 经过以下处理后作为输出文件名：
1. 若包含 ASCII 大写字母（camelCase），插入 `-` 并 lowercased（如 `myConfigName` → `my-config-name`）
2. 全小写或无大写字母的不变
3. 追加 `.json` 后缀
